# TinyMOA ISA Reference
> Covers the programmer model: registers, instruction encodings, and opcode tables. For RTL implementation details see Architecture.md.

## Extension Summary
Canonical status table: RV32I (full), E (x0–x15 only), C/Zca (full Q0/Q1/Q2), Zcb (full — C.MUL uses custom ALU opcode `4'b1010` instead of standard Zcb encoding), Zicond. No Zicsr. No M. No F. No TinyQV custom extensions.

| Extension            | Status                  | Notes                                 |
|----------------------|-------------------------|---------------------------------------|
| RV32I base           | Implemented             | Full                                  |
| E (embedded)         | Implemented             | 16 registers only, `x0` through `x15` |
| C / Zca (compressed) | Implemented             | Full Q0 / Q1 / Q2                     |
| Zcb                  | Implemented             | `c.lbu`, `c.lh`, `c.lhu`, `c.sb`, `c.sh`, `c.zext.b`, `c.sext.b`, `c.zext.h`, `c.sext.h`, `c.zext.w`, `c.mul`\*, `c.not` |
| Zicond               | Implemented             | `czero.eqz`, `czero.nez`              |
| M (multiply/divide)  | Not implemented         | Opcodes reserved                      |
| F (floating-point)   | Not implemented         | Opcodes reserved                      |
| CIM custom           | Considered, not planned | CIM via MMIO only                     |

\* **Note on `c.mul`**: The decoder implements the multiply instruction at quadrant 2, funct3=101 (`5'b10101`) with `alu_opcode=4'b1010`. This encoding does **not** match the standard Zcb `c.mul` (CA-type, Q1, funct6=100111). The current implementation is effectively a non-standard compressed multiply. Whether to fix the encoding to match the Zcb spec is an open question.


## Registers

TinyMOA uses RV32E with 16 registers (`x0–x15`) instead of the standard 32 to save physical space.

`gp` and `tp` are pseudo-hardcoded - their values are generated by combinational logic rather than stored in the register file, further saving space.

| Register  | ABI Name | Value / Role                                 | Saved By |
|-----------|----------|----------------------------------------------|----------|
| `x0`      | `zero`   | Hardwired 0, writes are ignored              | —        |
| `x1`      | `ra`     | Return address                               | Caller   |
| `x2`      | `sp`     | Stack pointer                                | Callee   |
| `x3`      | `gp`     | Global pointer (pseudo-hardcoded) `0x000400` | —        |
| `x4`      | `tp`     | Thread pointer (pseudo-hardcoded) `0x400000` | —        |
| `x5–x7`   | `t0–t2`  | Temporaries                                  | Caller   |
| `x8`      | `s0/fp`  | Saved register / frame pointer               | Callee   |
| `x9`      | `s1`     | Saved register                               | Callee   |
| `x10–x11` | `a0–a1`  | Arguments / return values                    | Caller   |
| `x12–x15` | `a2–a5`  | Arguments                                    | Caller   |

> *Caller*-saved: caller must preserve these across a call if it needs them.
> 
> *Callee*-saved: the called function must restore these before returning.

### Compressed Register Subset

Compressed instructions encode registers in 3 bits, limiting them to `x8–x15` and written with an apostrophe: `rd'`, `rs1'`, `rs2'`. Decode as `value + 8`.

| 3-bit encoding | Register | ABI Name |
|----------------|----------|----------|
| `000`          | `x8`     | `s0/fp`  |
| `001`          | `x9`     | `s1`     |
| `010`          | `x10`    | `a0`     |
| `011`          | `x11`    | `a1`     |
| `100`          | `x12`    | `a2`     |
| `101`          | `x13`    | `a3`     |
| `110`          | `x14`    | `a4`     |
| `111`          | `x15`    | `a5`     |


## Instruction Formats
R/I/S/B/U/J type bit layouts. Note: bits [1:0] are always `11` for 32-bit instructions (distinguishes from 16-bit compressed where [1:0] ≠ `11`). opcode field is bits [6:2] (5 significant bits).

### RV32I (Full 32-bit)

Opcode bits [1:0] are always `11` to denote 32-bit instructions.

| Type | Layout | Used For |
|------|--------|----------|
| **R** | `[funct7][rs2][rs1][funct3][rd][opcode]` | Register-register ops (ADD, SUB, AND…) |
| **I** | `[imm[11:0]][rs1][funct3][rd][opcode]` | Immediates, loads, JALR |
| **S** | `[imm[11:5]][rs2][rs1][funct3][imm[4:0]][opcode]` | Stores |
| **B** | `[imm[12\|10:5]][rs2][rs1][funct3][imm[4:1\|11]][opcode]` | Conditional branches |
| **U** | `[imm[31:12]][rd][opcode]` | Large constants (LUI, AUIPC) |
| **J** | `[imm[20\|10:1\|11\|19:12]][rd][opcode]` | Unconditional jumps (JAL) |

### RV32C (Compressed 16-bit)

Opcode bits [1:0] are either `00`, `01`, or `10` to denote which 16-bit quadrant (0-2) they belong to.

| Type    | Layout | Used For |
|---------|--------|----------|
| **CR**  | `[funct4][rd/rs1][rs2][op]` | Register ops (C.ADD, C.MV, C.JR) |
| **CI**  | `[funct3][imm][rd/rs1][imm][op]` | Immediate ops (C.ADDI, C.LI, C.LWSP) |
| **CSS** | `[funct3][imm][rs2][op]` | Stack-relative stores (C.SWSP) |
| **CIW** | `[funct3][imm][rd'][op]` | Wide immediate (C.ADDI4SPN) |
| **CL**  | `[funct3][imm][rs1'][imm][rd'][op]` | Loads (C.LW) |
| **CS**  | `[funct3][imm][rs1'][imm][rs2'][op]` | Stores (C.SW) |
| **CA**  | `[funct6][rd'/rs1'][funct2][rs2'][op]` | Arithmetic (C.SUB, C.AND, C.MUL) |
| **CB**  | `[funct3][offset][rs1'][offset][op]` | Branches (C.BEQZ, C.BNEZ) |
| **CJ**. | `[funct3][jump target][op]` | Jumps (C.J, C.JAL) |


## 32-bit Opcode Map
opcode[6:2] to instruction group. Standard RV32I only — no M/F/custom opcode groups.

### Internal ALU Opcodes

| Opcode | Operation | Source |
|--------|-----------|--------|
| `0000` | ADD | ADD, ADDI, AUIPC, address arithmetic |
| `0001` | SLL | SLL, SLLI |
| `0010` | SLT (signed) | SLT, SLTI, BLT, BGE |
| `0011` | SLTU (unsigned) | SLTU, SLTIU, BLTU, BGEU |
| `0100` | XOR | XOR, XORI, BEQ, BNE, C.NOT |
| `0101` | SRL | SRL, SRLI |
| `0110` | OR | OR, ORI |
| `0111` | AND | AND, ANDI, C.ZEXT.* |
| `1000` | SUB | SUB, C.SUB |
| `1010` | MUL | C.MUL\* |
| `1101` | SRA | SRA, SRAI |
| `1110` | CZERO.EQZ | Conditional zero if rs2 == 0 |
| `1111` | CZERO.NEZ | Conditional zero if rs2 != 0 |

\* This C.MUL is not a standard Zcb extension opcode, additionally, it is only 16-bit X 16-bit, to produce a 32-bit product.

### Internal Memory Opcodes
3-bit dec_mem_opcode: size (byte/half/word) and unsigned flag.

## 16-bit Compressed Instructions
Full Q0/Q1/Q2 tables. Zcb byte/halfword loads and stores fully implemented. C.MUL: rd' = rd'[15:0] × rs2'[15:0], result truncated to 32 bits.

# TinyMOA ISA Reference

TinyMOA implements `RV32EC` with extensions `Zcb`, `Zicsr`, `Zicond`. No custom CIM instructions exist, it is controlled via MMIO.

## Extension Summary

| Extension | Status | Notes |
|-----------|--------|-------|
| RV32I base | Implemented | Full |
| E (embedded) | Implemented | 16 registers only, `x0` through `x15` |
| C / Zca (compressed) | Implemented | Full Q0 / Q1 / Q2 |
| Zcb | Implemented | `c.lbu`, `c.lh`, `c.lhu`, `c.sb`, `c.sh`, `c.zext.b`, `c.sext.b`, `c.zext.h`, `c.sext.h`, `c.zext.w`, `c.mul`\*, `c.not` |
| Zicond | Implemented | `czero.eqz`, `czero.nez` |
| M (multiply/divide) | Not implemented | Opcodes reserved |
| F (floating-point) | Not implemented | Opcodes reserved |
| CIM custom | Not planned | CIM via MMIO only |

\* **Note on `c.mul`**: The decoder implements the multiply instruction at quadrant 2, funct3=101 (`5'b10101`) with `alu_opcode=4'b1010`. This encoding does **not** match the standard Zcb `c.mul` (CA-type, Q1, funct6=100111). The current implementation is effectively a non-standard compressed multiply. Whether to fix the encoding to match the Zcb spec is an open question.


## Registers

| Register | ABI | Physical value | Role |
|----------|-----|----------------|------|
| `x0` | `zero` | 0 | Hardwired zero |
| `x1` | `ra` | — | Return address (caller-saved) |
| `x2` | `sp` | — | Stack pointer (callee-saved) |
| `x3` | `gp` | `0x000400` | Global pointer — pseudo-hardcoded |
| `x4` | `tp` | `0x400000` | Thread/peripheral pointer — pseudo-hardcoded |
| `x5–x7` | `t0–t2` | — | Temporaries (caller-saved) |
| `x8` | `s0/fp` | — | Saved / frame pointer (callee-saved) |
| `x9` | `s1` | — | Saved register (callee-saved) |
| `x10–x11` | `a0–a1` | — | Arguments / return values (caller-saved) |
| `x12–x15` | `a2–a5` | — | Arguments (caller-saved) |

`gp` and `tp` do not consume register file storage. Their values are generated by combinational logic in `registers.v` using `nibble_counter` to output the correct nibble each cycle. `gp=0x000400` is the SRAM midpoint (used for gp-relative data access). `tp=0x400000` is the peripheral base (CIM MMIO, used for tp-relative CIM access).

### Compressed Register Subset

Compressed instructions use 3-bit register fields, mapping to `x8–x15` (`value + 8`).

| 3-bit | Register | ABI   |
|-------|----------|-------|
| 000   | x8       | s0/fp |
| 001   | x9       | s1    |
| 010   | x10      | a0    |
| 011   | x11      | a1    |
| 100   | x12      | a2    |
| 101   | x13      | a3    |
| 110   | x14      | a4    |
| 111   | x15      | a5    |

## 32-bit Instruction Formats

| Type | Bit layout | Used for |
|------|-----------|---------|
| R    | `[31:25 funct7][24:20 rs2][19:15 rs1][14:12 funct3][11:7 rd][6:0 opcode]` | Register-register |
| I    | `[31:20 imm[11:0]][19:15 rs1][14:12 funct3][11:7 rd][6:0 opcode]` | Loads, immediates, JALR |
| S    | `[31:25 imm[11:5]][24:20 rs2][19:15 rs1][14:12 funct3][11:7 imm[4:0]][6:0 opcode]` | Stores |
| B    | `[31 imm[12]][30:25 imm[10:5]][24:20 rs2][19:15 rs1][14:12 funct3][11:8 imm[4:1]][7 imm[11]][6:0 opcode]` | Branches |
| U    | `[31:12 imm[31:12]][11:7 rd][6:0 opcode]` | LUI, AUIPC |
| J    | `[31 imm[20]][30:21 imm[10:1]][20 imm[11]][19:12 imm[19:12]][11:7 rd][6:0 opcode]` | JAL |

## Memory Opcode (3-bit, `dec_mem_opcode`)

| `[2:0]` | Meaning |
|---------|---------|
| `000` | LB / SB — byte, sign-extend on load |
| `001` | LH / SH — halfword, sign-extend on load |
| `010` | LW / SW — word |
| `100` | LBU - byte, zero-extend |
| `101` | LHU - halfword, zero-extend |

`mem_opcode[1:0]` = size (byte/half/word). `mem_opcode[2]` = unsigned flag (U).
For branches, `mem_opcode[0]` = polarity invert (BNE/BGE/BGEU vs BEQ/BLT/BLTU).

## 16-bit Compressed Instructions

### Quadrant 00 (`instr[1:0] = 00`)

| Encoding | Mnemonic | Operation |
|----------|----------|-----------|
| funct3=000 | C.ADDI4SPN | rd' = sp + nzuimm×4 |
| funct3=010 | C.LW | rd' = mem[rs1'+offset] |
| funct3=100 | C.LBU/LHU/LH/SB/SH | Byte/halfword load-store |
| funct3=110 | C.SW | mem[rs1'+offset] = rs2' |

### Quadrant 01 (`instr[1:0] = 01`)

| Encoding | Mnemonic | Operation |
|----------|----------|-----------|
| funct3=000 | C.ADDI / C.NOP | rd += nzimm |
| funct3=001 | C.JAL | ra = PC+2; PC += offset |
| funct3=010 | C.LI | rd = imm |
| funct3=011 | C.ADDI16SP / C.LUI | sp += nzimm×16 or rd = imm<<12 |
| funct3=100, instr[11:10]=00 | C.SRLI | rd' >>= uimm (logical) |
| funct3=100, instr[11:10]=01 | C.SRAI | rd' >>= uimm (arithmetic) |
| funct3=100, instr[11:10]=10 | C.ANDI | rd' &= imm |
| funct3=100, instr[12]=1 | C.NOT / C.ZEXT.* | Bitwise NOT or zero-extend |
| funct3=100, instr[12]=0 | C.SUB/XOR/OR/AND | Register arithmetic |
| funct3=101 | C.J | PC += offset |
| funct3=110 | C.BEQZ | if (rs1' == 0) PC += offset |
| funct3=111 | C.BNEZ | if (rs1' != 0) PC += offset |

### Quadrant 10 (`instr[1:0] = 10`)

| Encoding | Mnemonic | Operation |
|----------|----------|-----------|
| funct3=000 | C.SLLI | rd <<= uimm |
| funct3=100, instr[12]=0, rs2=0 | C.JR | PC = rs1 |
| funct3=100, instr[12]=0, rs2≠0 | C.MV | rd = rs2 |
| funct3=100, instr[12]=1, rs1≠0, rs2≠0 | C.ADD | rd += rs2 |
| funct3=100, instr[12]=1, rs1≠0, rs2=0 | C.JALR | ra = PC+2; PC = rs1 |
| funct3=100, instr[12]=1, rs1=0, rs2=0 | C.EBREAK | Halt |
| funct3=101 | C.MUL | rd' *= rs2'[15:0] (non-standard encoding, see note) |
| funct3=110 | C.SWSP | mem[sp + offset] = rs2 |
| funct3=111 | C.SWTP | mem[tp + offset] = rs2 |
